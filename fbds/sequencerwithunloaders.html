SequencerWithUnloaders

Compressor Control Advanced FunctionBlock v1.2.3

Description

This block manages the outputs of compressors (or partialization of compressor) respecting the safe time of each compressor and with synchronism by additional signals.

To turn on the compressor is necessary to have a proper rPowerRequest, in case of 2 compressors it must be at least 50%, in case of 3 at least 33.4%, 25% if 4...
Then also the xEnInc must be TRUE and bEnOnOff must be 1.

At the same way, to swith Off one compressor the input xEnDec must be TRUE, to switch Off the last compressor the usiEnOnOff must be 2.

The enabling xEnInc for INCresing number of compressors as xEnDec for DECreasing is useful in case of multicircuit system, to allow the turning on or off of one compressor per time. The outputs xReqInc and xReqDec have the function of indicating the possibility to switch on and off respectively, to send to external semaphore or other block to distribute these request, in case of only one circuit, the outputs xReqInc and xReqDec can be connected directly or by additional variables to inputs xEnInc and xEnDec.

The output usiReqOnOff can be connected to input usiEnOnOff if no additional devices are involved in regulation. Fot example, in presence of Liquid Valve, before turning on the compressor, the liquid valve must be On for some time in advance, so the usiReqOnOff can be connected to Liquid Valve block, then from Liquid Valve block a specified output toward the input of sequencer xEnOnOff will allow the start of compressor.
In the same way, the Liquid Valve could be switched off before last compressor, in this case the sequencer can switch off the last compressor only if bEnOnOff = 2.

So, about the input usiEnOnOff:
- if 0 no changes in the outputs are allowed;
- if 1 it is possible to switch on the compressors and also to switch off with exception of last compressor;
- if 2 also the last compressor can be switched off.
