VZH_HeatsinkTemperatureControl

Compressor Control Advanced FunctionBlock v1.2.4

FBD Diagram

vzh_heatsinktemperaturecontrol.jpg

Description



The brick controls the heatsink temperature, it provides to set warning and alarm if respective thresholds are over. It can reduce the request to compressor in case of temperature heatsink in warning area.

If xEnable is FALSE or the xComprRun is FALSE, the brick is disable, all its outputs are FALSE or 0.

If xEnable is TRUE and xComprRun is TRUE, then rHeatsinkTemperature (heatsink temperature) is comparated with reference setpoints, in particular rHeatsinkWarningLevel and rHeatsinkEmergencyLevel, both with the same differential rHeatsinkDifferential.
If rHeatsinkTemperature is above rHeatsinkEmergencyLevel the output xHeatsinkAlarmStatus is set, it is reset when the temperature goes below rHeatsinkEmergencyLevel - rHeatsinkDifferential.
If xHeatsinkWarningInput is FALSE and rHeatsinkTemperature is above rHeatsinkWarningLevel the output xHeatsinkWarningStatus is set, it is reset when the temperature goes below rHeatsinkWarningLevel - rHeatsinkDifferential.
If xHeatsinkWarningInput is TRUE (this signal should be a warning from inverter) then the xHeatsinkWarningStatus assumes the value of xHeatsinkDriveWarningStatus.
If the probe for rHeatsinkTemperature is in fault, the input xHeatsinkTemperatureFault should be TRUE, in this case the output xHeatsinkAlarmStatus is TRUE.

It is possible to manage the demand toward the compressor if the input xHeatsinkWarningAction = TRUE. In this case if xHeatsinkWarningStatus = TRUE then the rComprRequestOut is reduced starting from rComprActualPercent to at min rComprMinSpeedPercent, in step of rHeatsinkStepValue every uiHeatsinkStepTime.
When the xHeatsinkWarningStatus becomes FALSE, then the rComprRequestOut has increased until rComprRequestIn at rHeatsinkStepValue every uiHeatsinkStepTime.
If xHeatsinkWarningInput is TRUE (this signal should be a warning from inverter) then the xHeatsinkWarningStatus assumes the value of xHeatsinkDriveWarningStatus, if it is FALSE the rComprRequestOut increases, but if the xHeatsinkAlarmStatus is TRUE then the rComprRequestOut continues to decrease.
During the managing of compressor request the output xStatus is TRUE.
