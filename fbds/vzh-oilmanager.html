Description



The function block controls the oil level in VZH compressors and takes the appropriate actions to ensure the safe operation of the compressors. It tries first to push the oil trapped in the system back to compressor, increasing the compressor speed for short periods (boost procedure) and finally it generates an alarm when necessary.

There are four different modes to operate, depending from the value of usiOilBoostType:
- if 0 the boost happens in TIME, if rComprActualPercent is below rMinSpeedLimitPercent for uiLowSpeedRunningTime minutes, then the output xOilBoostStatus becomes TRUE for uiBoostDuration. With period uiFixedBoostInterval the operation is repeated, this time is reset if the rComprActualPercent goes above rOilBoostSpeedPercent.

- if 1 the boost happens in LEVEL, if input xLackOfOilSignal is TRUE (means the optical or equivalent sensor detect missing of oil) for at least uiLackOfOilDelay, then the outputs xOilBoostStatus becomes TRUE, the rOilBoostSpeedPercent assumes the value defined by rBoostDemandPercent, only when the rComprActualPercent reaches at least rBoostPercent -0.5%, then the uiBoostDuration time starts to count down, when elapsed the boost is completed, xOilBoostStatus goes to FALSE and rOilBoostSpeedPercent to 0.


- if 2 the boost happens in TIME and LEVEL, as described before.
- if 3 the boost happens in LEVEL but only for alarms generation, the output xOilBoostStatus stays FALSE and rOilBoostSpeedPercent =0.

The output xLackOfOilStatus becomes TRUE when the input xLackOfOilSignal is TRUE for at least uiLackOfOilDelay seconds, becomes FALSE if xLackOfOilSignal is FALSE for uiRecoveryOfOilDelay seconds.

Alarms/Warning
- xStartWithoutOilAlarm becomes TRUE if the LEVEL is TRUE at starting of compressor (xComprRun becomes TRUE) until uiStartWithoutOilWindow seconds, and the LEVEL is not recovered within uiStartPumpOutTime seconds.
- xLackOfOilAlarm becomes TRUE if , after the StartWithoutOilWindow time, the LEVEL is not recovered within uiPumpOutTime seconds.
- xTimeBetween_2_LackOfOilWarning becomes TRUE if the LEVEL is TRUE for 2 times during uiIntervalBetween_2_LackOfOilWarning minutes.
- xTimeBetween_2_LackOfOilAlarm becomes TRUE if the LEVEL is TRUE for 2 times during uiIntervalBetween_2_LackOfOilAlarm minutes.


Timers
There are some outputs to show the timer values counting, useful for all scopes.
