ThreePtsValve

Valve Control Intermediate FunctionBlock v1.2.3

FBD Diagram

threeptsvalve.png

Description



At the beginning, independently from xEnable, the xClose is actived for uiValvePeriod + uiValvePeriod/4 to guarantee the closing position of valve.

If xEnable is TRUE, and xFullClose is TRUE the valve is forced to close for uiValvePeriod + uiValvePeriod/4 (overdrive), with priority compared with xFullOpen.

If xEnable is TRUE, and xFullOpen is TRUE the valve is forced to open for uiValvePeriod + uiValvePeriod/4.

If xEnable is TRUE, xFullOpen = FALSE and xFullClose = FALSE then: 1) At the beginning the valve is forced to close for uiValvePeriod + uiValvePeriod/4 (overdrive);
2) if the position of valve rValvePos is 0%, every 10 minutes the output xClose becomes TRUE for the time uiValvePeriod/4;
3) if the position of valve rValvePos is 100%, every 10 minutes the output xOpen becomes TRUE for the time uiValvePeriod/4;
Note:option 2) and 3) overdrive the valve to allow alignment of estimated position to real position. 4) if rPowReq is greater than previous rPowReq above rSensitivity then the output xOpen is activated for the time defined by the formula:

time OPEN [s] = (rPowReq - rValvePos) * uiValvePeriod/100

5) if rPowReq is lower than previous rPowReq above rSensitivity then the output xClose is actived for the time defined by:

time CLOSE [s] = (rPowReq - rValvePos) * uiValvePeriod/100;

Note: option 4) and 5) have an exception when rPowReq is lower than 10% or greater than 90%, in these cases the rPowReq maintains the value. 6) if rPowReq is around rValvePos within rSensistivity then the outputs xClose and aOpen are not activated.
