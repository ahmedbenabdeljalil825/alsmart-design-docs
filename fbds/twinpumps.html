TwinPumps

Pump Control Intermediate FunctionBlock v1.2.3

FBD Diagram

imgtwinpumps.png (Icon)

twinpumps.png

Description



If xEnable is FALSE, then the outputs xStatus, xPump1 and xPump2 are FALSE.

If xEnable is TRUE, there are the following cases:
1) If number of pumps bNumOfPumps is different from 1 or 2, then the outputs xStatus, xPump1 and xPump2 are FALSE.
2) If bNumOfPumps =1, then the outputs xStatus, xPump1 becomes TRUE if xPumpRequest = TRUE and xPump1Alarm = FALSE.
3) If bNumOfPumps = 2, and bPumpSelection = 0, then the pumps xPump1 and xPump2 are alternatively turned ON, when xPumpRequest becomes TRUE and they are not in alarm.
In particular, the first time when xPumpRequest becomes TRUE then xPump1 switch ON (TRUE), then when xPumpRequest = FALSE the xPump1 switches OFF (FALSE), the next time xPumpRequest becomes TRUE then xPump2 becomes TRUE ...
4) If the bNumOfPumps = 2, and bPumpSelection = 1, then the pump xPump1 is turned ON when xPumpRequest becomes TRUE, and its alarm xPump1Alarm = FALSE, if this alarm is TRUE then the xPump2 is turned ON.
5) If the bNumOfPumps = 2, and bPumpSelection = 2, then the pump xPump2 is turned ON when xPumpRequest becomes TRUE, and its alarm xPump2Alarm = FALSE, if this alarm is TRUE then the xPump1 is turned ON.

Inputs (6)

Name Type Init Description

bNumOfPumps BYTE BYTE, range [1,2]
xPump1Alarm BOOL BOOLEAN, range [FALSE,TRUE]
xPump2Alarm BOOL BOOLEAN, range [FALSE,TRUE]
bPumpSelection INT BYTE, range [0,2]. If equal to 0 the pumps are alternatevely switched ON, if 1 then PUMP_1 is switched ON, 2 for PUMP_2
