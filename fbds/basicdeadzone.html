BasicDeadZone

Control Logic Advanced FunctionBlock v1.2.3

FBD Diagram

basicdeadzone.png

Description



With xDirection = FALSE the mode is COOLING or COOL, rOut increases with increasing of rIn above rSetpoint + rDeadZone,
with xDirection = TRUE the mode is HEATING or HEAT, rOut increases with decresing of rIn below rSetpoint - rDeadZone.

If xEnable is FALSE the rOut is 0, uiCountDown is uiInterval, InDeadZone is FALSE.

If xEnable is TRUE, every uiInterval secs the rOut is updated in following way:
1) if rIn > rSetpoint + rDeadZone and xDirection = FALSE then rOut increases of rDelta, if xDirection = TRUE then rOut decreases of rDelta;
2) if rIn < rSetpoint - rDelatZone and xDirection = TRUE then rOut increases of rDelta, if xDirection = FALSE then rOut decreases of rDelta;
3) if rIn is between rSetpoint - rDeltaZone and rSetpoint + rDeltaZone the rOut maintains the previous value.

The rOut limits are between rMinOut and rMaxOut.

uiCountDown shows the counting down of uiInterval, when it reaches 0 then rOut is updated.

If xInDeadZone is TRUE it means that rIn is between rSetpoint - rDeadZone and rSetpoint + rDeadZone .

If xEnableResetOut is TRUE the rOut assumes the value of rResetOutValue.
