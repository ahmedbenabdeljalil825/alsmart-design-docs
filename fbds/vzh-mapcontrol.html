Description



The function block controls the saturated evaporating and condensing temperature values and check that the VZH compressor working zone is inside the compressor envelope. Based on the working zone, a new compressor demand is generated with the target to keep the compressor working in a safety zone. A warning or alarm message is generated when needed. Then it is possible to reduce the demand to compressor in case of discharge temperature in warning area. The following range of VZH compressors is managed: • VZH088-117-170 High PR • VZH088-117-170 version C • VZH052-065 • VZH028-035-044 version B • ... custom compressor map ... The compressor envelopes or maps are defined in the VZH_CompressorMaps.pll file which is part of this brick. It is possible to update this file adding custom map of specific compressor or modifing the existing, the unique constrain is to maintain the number of type in 5, each one with 15 polygones.

How to fill the Polygones information CompressorMaps[i,j].nNumVertices := number of vertices of polygon
CompressorMaps[i,j].iPolyPoints[i] := x,y coordinates, if i is even number is x cordinate, if odd is y

x represents the Evaporator Temperature in °Cx10
x represents the Condenser Temperature in °Cx10

i.e.:
CompressorMaps[i,j].iPolyPoints[0] := x coordinate of first vertice
CompressorMaps[i,j].iPolyPoints[1] := y coordinate of first vertice
CompressorMaps[i,j].iPolyPoints[5] := y coordinate of third vertice

Then, there are additional information specific for each zone (polygon) individuated, in particular:
• uiTimeoutSec := max time in seconds allowed in zone
• iRampStep := value of speed rate change defined for the zone when siActionType= 1 or 2. This is reserved for future use as it is recommended to use the ramp defined in the compressor drive.
• siActionType:= type of action when the compressor is working in that zone, in particular:
-1 = no action
0 = keep speed between min speed and max speed, no time limits
1 = increase speed to max speed (and always within speed limits), limited time allowed in zone (uiTimeoutSec)
2 = decrease speed to min speed (and always within speed limits), limited time allowed in zone (uiTimeoutSec)
3 = keep speed between min speed and max speed, limited time allowed in zone (uiTimeoutSec)
4 = if MOP Disabled: increase speed to max speed (and always within speed limits);
if MOP Enabled: keep speed between min speed and max speed;
limited time allowed in zone (uiTimeoutSec)
5 = if LOP Disabled: decrease speed to min speed (and always within speed limits);
if LOP Enabled: keep speed between min speed and max speed;
limited time allowed in zone (uiTimeoutSec)
• uiMinSpeed := Minimum compressor speed in RPS allowed in zone
• uiMaxSpeed := Maximum compressor speed in RPS allowed in zone
• usiAlarmType := type of alarm:
0 = none
1 = the output xSafetyShutdownAlarm is activated without any delay if the compressor is working in that zone (typically the compressor must be stopped)
2 = the output xTimerShutdownAlarm is activated if the compressor is working in that zone for more than timeout seconds
• usiAlarmCode := code of alarm:
0-4 = code alarm
5 = it is the output of the brick if no zone (0) has been individued with xSafetyShutdownAlarm or xTimerShutdownAlarm TRUE.

Two more actions are requested after the selection of FB:
1) With right button of mouse, select Imports Object and select the file VZH_CompressorMaps.pll,
then select InizializeCompressorMaps, press Import Objects and Close;
2) Then in Tasks, select Init and add InizializeCompressorMaps.

Outputs
- usiZone = it is the number of polygon individuated by rEvapTemp and rCondTemp coordinates, for compressor index usiComprType.
- rRampStep = it is the filed rRampStep of usiZone(polygon) individuated.
- usiAlarmCode = it is the field usiAlarmCode of usiZone (polygon) individuated if alarms xTimerShutdownAlarm or xSafetyShutdownAlarm is TRUE , it is 5 in case of usiZone = 0.
- xSafetyShutdownAlarm= it becomes TRUE if usiZone is 0 and xComprOff is FALSE.
- xTimerShutdownAlarm = it becomes TRUE if the compressor remains in usiZone more time of uiTimeoutSec defined in the map.
- xDT_EmergencyAlarm = it becomes TRUE when the rDischargeTempValue goes above rDT_EmergencyLevel, it becomes FALSE when the value goes below rDT_EmergencyLevel-rDT_Differential.
- xDT_LevelWarning = it becomes TRUE when the rDischargeTempValue goes above rDT_WarningLevel, it becomes FALSE when the value goes below rDT_WarningLevel-rDT_Differential.
- rComprPercent = it assumes the value between min and max of the map, but this value is reduced if xDT_LevelWarning is TRUE and the input xDT_WarningAction = TRUE; in this case the rComprPercent is reduced at min defined in the map in step of rDT_StepValue every uiDT_StepTime.
Then, if xDT_LevelWarning becomes FALSE, then the rComprPercent has increased until to max defined in the map at rDT_StepValue every uiDT_StepTime.
