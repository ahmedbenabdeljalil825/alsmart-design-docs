BandDeadZone

Control Logic Advanced FunctionBlock v1.2.3

Description



If xEnable is FALSE the rOut is 0.



If xEnable is TRUE, then rOut is updated in following way:
1) if rIn is between rSetpoint and rSetpoint + rDeadZone the rOut maintains the previous value;
2) if rIn > rSetpoint + rDeadZone then rOut increases of 100.0/bNumOfStages every time period depending from how far is from rSetpoint + rDeadZone.
In particular this time is proportional to rIn versus rIncBand, and uiMinIncDelay versus uiMaxIncDelay, i.e.
if rIn is just above rSetpoint + rDeadZone then rOut increases every uiMaxIncDelay,
else if rIn is at or above rSetpoint + rDeadZone + rIncBand then the delay is uiMinIncDelay;
if rIn is in the middle of rIncBand then, with the same proportion, this time is between uiMinIncDelay and uiMaxIncDelay.



3) if rIn < rSetpoint then rOut decreases of 100.0/bNumOfStages every time period depending from how far is from rSetpoint.
In particular this time is proportional to rIn versus rDecBand, and uiMinDecDelay versus uiMaxDecDelay, i.e.
if rIn is just below rSetpoint then rOut decreases every uiMaxDecDelay,
else if rIn is to or below rSetpoint - rDecBand then the delay is uiMinDecDelay;



4) if rIn is below rOffValue then rOut is 0.
